# Taxonomy

The domains you actually think and talk about. Synthesis files each note into one of these
first. Add, rename, or remove domains freely — this is yours to shape.

## Format

- Each `## Heading` is a domain.
- The lines under it describe the domain and (optionally) list example `[[notes]]`.
- The app never invents a domain. When it hears something that fits nowhere, it appends a
  suggestion under `## Proposed` for you to promote (or delete).

---

## Software

Concepts, patterns, protocols, and tooling — the durable knowledge, not a specific project.
Examples: [[Domain-Driven Design]], [[OAuth 2.0]], [[OpenID Connect]], [[Authentication]]

## Projects

Things you're actively building, with roadmaps and decisions. Project notes link out to the
Software concepts they use.
Examples: [[Bellhop]]

## Learning

Resources you're working through, open questions, and "I didn't realize that" moments worth
revisiting. Often links to the Software concept it clarifies.
Examples: [[OAuth 2.0]]
